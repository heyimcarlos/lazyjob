Sub-view Pattern

How job detail lives inside the Jobs tab

The Problem

ViewId is a simple enum (Copy, no payload) used for tab navigation (1-6 keys). Adding ViewId::JobDetail would break tab_index() and complicate the header tabs.

The Solution

A viewing_job_detail: bool flag on App:

pub fn active_view_mut(&mut self) -> &mut dyn View {
    match self.active_view {
        ViewId::Jobs => {
            if self.viewing_job_detail {
                &mut self.views.job_detail
            } else {
                &mut self.views.jobs_list
            }
        }
        // ... other views
    }
}

Navigation Flow

  • Enter on job -> OpenJob(id) -> sets viewing_job_detail = true
  • Esc -> NavigateBack -> clears viewing_job_detail (stays on Jobs tab)
  • Tab switch (1-6) -> NavigateTo -> clears viewing_job_detail
  • Header tab stays highlighted on "Jobs" the whole time

Salary Formatting

fn format_salary(min: Option<i64>, max: Option<i64>) -> String {
    match (min, max) {
        (Some(lo), Some(hi)) => "$120k - $180k",
        (Some(lo), None)    => "$120k+",
        (None, Some(hi))    => "Up to $180k",
        (None, None)        => "--",
    }
}

Description Scrolling

Smooth scrolling through long job descriptions

Implementation

Uses ratatui's built-in Paragraph scroll support:

let desc_para = Paragraph::new(description)
    .wrap(Wrap { trim: false })
    .scroll((self.scroll_offset, 0))
    .block(Block::default()
        .title(" Description "));

Scroll Controls

KeyAction
j / DownScroll down one line
k / UpScroll up one line (clamped to 0)

Scroll offset resets to 0 when a new job is loaded via set_job().

  • Action bar text changes based on application state

New Action Variants

4 new actions added to the Action enum

pub enum Action {
    // ... existing variants ...
    ApplyToJob(JobId),         // a key - future task
    TailorResume(JobId),       // r key - future task
    GenerateCoverLetter(JobId),// c key - future task
    OpenUrl(String),           // o key - opens browser
}

OpenUrl Implementation

Uses the open crate (v5) for cross-platform URL opening:

Action::OpenUrl(url) => {
    let _ = open::that(&url);
}

Fire-and-forget: errors are silently ignored (browser may not be available).
