Task 18 — Protocol Types

WorkerCommand & WorkerEvent

Tagged union enums with serde snake_case serialization

WorkerCommand (TUI → subprocess)

#[derive(Debug, Clone, Serialize, Deserialize, PartialEq)]
#[serde(tag = "type", rename_all = "snake_case")]
pub enum WorkerCommand {
    Start {
        loop_type: String,       // "job_discovery", "resume_tailor", etc.
        params: serde_json::Value, // Arbitrary per-loop config
    },
    Cancel,
}

Wire format examples:

// Start
{"type":"start", "loop_type":"resume_tailor", "params":{"job_id":"abc-123"}}

// Cancel
{"type":"cancel"}

WorkerEvent (subprocess → TUI)

#[derive(Debug, Clone, Serialize, Deserialize, PartialEq)]
#[serde(tag = "type", rename_all = "snake_case")]
pub enum WorkerEvent {
    Status  { phase: String, progress: f32, message: String },
    Results { data: serde_json::Value },
    Error   { code: String, message: String },
    Done    { success: bool },
}

Wire format examples:

// Status update (progress bars in RalphPanel)
{"type":"status", "phase":"analyzing", "progress":0.42, "message":"Matching skills"}

// Results (output path, stats, etc.)
{"type":"results", "data":{"resume_path":"~/.lazyjob/output/acme-resume.docx"}}

// Error (retryable or fatal)
{"type":"error", "code":"api_timeout", "message":"Request timed out after 30s"}

// Done
{"type":"done", "success":true}

Error Type

#[derive(Debug, Error)]
pub enum RalphError {
    #[error("decode error: {0}")]
    Decode(String),
}

pub type Result<T> = std::result::Result<T, RalphError>;

Task 18 — NdjsonCodec

NdjsonCodec

Zero-size struct with associated encode/decode functions

Implementation

pub struct NdjsonCodec;

impl NdjsonCodec {
    // Infallible — WorkerCommand is always serializable
    pub fn encode(cmd: &WorkerCommand) -> String {
        let mut s = serde_json::to_string(cmd)
            .expect("WorkerCommand serialization never fails");
        s.push('\n');
        s
    }

    // Fallible — user-provided lines may be invalid JSON
    pub fn decode(line: &str) -> Result<WorkerEvent> {
        serde_json::from_str(line.trim())
            .map_err(|e| RalphError::Decode(e.to_string()))
    }
}

Design Notes

  • encode is infallible — WorkerCommand contains only serializable types (String, f32, bool, Value). Returning String directly is ergonomic and correct.
  • decode trims whitespace — BufReader::read_line() includes the trailing \n; trim() handles this without requiring callers to strip it.
  • Unknown type tags return Err — serde's tagged enum rejects unknown variants, which is tested explicitly.
  • Zero-size struct — no state needed; all methods are pure functions on their inputs.

Usage Pattern (task 19)

// TUI → subprocess stdin
let line = NdjsonCodec::encode(&WorkerCommand::Start {
    loop_type: "resume_tailor".to_string(),
    params: serde_json::json!({ "job_id": job_id }),
});
child_stdin.write_all(line.as_bytes()).await?;

// subprocess stdout → TUI broadcast
while let Some(Ok(line)) = stdout_lines.next_line().await {
    match NdjsonCodec::decode(&line) {
        Ok(event) => event_tx.send(event)?,
        Err(e) => tracing::warn!("bad event line: {e}"),
    }
}

Task 18 — Design Decisions

Design Decisions

Key choices and the reasoning behind them

Decision Chosen Why
Type names WorkerCommand / WorkerEvent Task description takes priority over spec's IncomingMessage / OutgoingMessage. Worker* naming makes direction explicit.
loop_type field type String Task 20 defines the LoopType enum. Using String avoids coupling to an unimplemented type while remaining forward-compatible.
encode return type String (infallible) WorkerCommand only contains serializable primitives. Infallible encode removes error handling boilerplate at every call site.
decode whitespace handling line.trim() before parse tokio::io::BufReader::lines() strips \n but read_line() does not. trim() handles both cases without extra branching.
RalphError::Decode wraps String No From<serde_json::Error> Avoids implicit ? conversions from serde in non-decode contexts. Explicit map_err is intentional — decode errors need context about what line failed.
PartialEq on enums Derived Required for test assertions (assert_eq!). serde_json::Value already implements PartialEq.

Forward Compatibility

When Task 20 (ralph-loop-types) adds the LoopType enum, the protocol module can optionally deserialize loop_type: String into it via loop_type.parse::<LoopType>() in the subprocess handler — no breaking changes to the wire format.

Relationship to Spec

The spec (specs/06-ralph-loop-integration-implementation-plan.md) defined IncomingMessage / OutgoingMessage with more variants (Pause, Resume). The task description is more focused: WorkerCommand::Start + Cancel only. The task description wins — keeping the protocol minimal until the process manager (Task 19) needs more variants.
