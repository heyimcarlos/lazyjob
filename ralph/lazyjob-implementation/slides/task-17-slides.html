03 · cost.rs

Cost Estimation

Microdollar pricing: 1 microdollar = $0.000001. The table stores blended input/output rates per 1,000 tokens. Matching is done by substring so model variants like claude-haiku-4-5-20251001 match the "claude-haiku" prefix.

// Microdollar pricing per 1000 tokens (blended input/output)
const PRICING: &[(&str, u64)] = &[
    ("claude-opus",  45_000),   // ~$45 per 1M tokens
    ("claude-sonnet", 9_000),   // ~$9 per 1M tokens
    ("claude-haiku",    750),   // ~$0.75 per 1M tokens
    ("gpt-4o-mini",     375),   // ~$0.375 per 1M tokens
    ("gpt-4o",        10_000), // ~$10 per 1M tokens
    ("gpt-4",         45_000), // ~$45 per 1M tokens
    ("gpt-3.5",        1_000), // ~$1 per 1M tokens
    ("llama",               0),   // free (local)
    ("mistral",             0),   // free (local)
    ("nomic",               0),   // free (local)
];

pub fn estimate_cost(model: &str, tokens: u32) -> u64 {
    let rate = PRICING
        .iter()
        .find(|(prefix, _)| model.contains(prefix))
        .map(|(_, rate)| *rate)
        .unwrap_or(DEFAULT_RATE);
    (tokens as u64).saturating_mul(rate) / 1_000
}

04 · ProviderRegistry

Provider Registry

A named container for Arc<dyn LlmProvider> instances. The first provider added automatically becomes the default; you can override with set_default().

pub struct ProviderRegistry {
    providers: HashMap<String, Arc<dyn LlmProvider>>,
    default: Option<String>,
}

impl ProviderRegistry {
    pub fn new() -> Self
    pub fn add(&mut self, name: impl Into<String>, provider: Arc<dyn LlmProvider>)
    pub fn get(&self, name: &str) -> Option<Arc<dyn LlmProvider>>
    pub fn default_provider(&self) -> Option<Arc<dyn LlmProvider>>
    pub fn all(&self) -> Vec<(String, Arc<dyn LlmProvider>)>
    pub fn set_default(&mut self, name: impl Into<String>)
}

Usage Pattern

let mut reg = ProviderRegistry::new();
reg.add("anthropic", Arc::new(AnthropicProvider::new(key)));
reg.add("ollama",    Arc::new(OllamaProvider::new()));

// First added (anthropic) is default:
let provider = reg.default_provider().unwrap();
let resp = provider.complete(messages, opts).await?;
Why Arc<dyn LlmProvider>?

Allows cheap cloning of provider handles. Multiple components (TUI panel, CLI, Ralph loops) can hold a reference to the same provider without duplication.

Future Extension

Tasks 22-36 (Ralph loops, resume tailoring, cover letters) will use ProviderRegistry to pick the appropriate provider per operation type.

05 · LlmBuilder

LlmBuilder::from_config

Builds the best available provider from Config and CredentialManager — never panics, never errors. When all else fails it returns an Ollama provider (always available locally).

pub fn from_config(config: &Config, creds: &CredentialManager)
    -> Result<Box<dyn LlmProvider>>
{
    // 1. Try the configured provider first
    if let Some(ref name) = config.default_llm_provider {
        match name.as_str() {
            "anthropic" => if let Ok(p) = AnthropicProvider::from_credentials(creds) {
                return Ok(Box::new(p));
            }
            "ollama"    => return Ok(Box::new(OllamaProvider::new())),
            _ => {}
        }
    }
    // 2. Try Anthropic, then OpenAI
    if let Ok(p) = AnthropicProvider::from_credentials(creds) { return Ok(Box::new(p)); }
    if let Ok(p) = OpenAiProvider::from_credentials(creds)   { return Ok(Box::new(p)); }
    // 3. Always-available fallback
    Ok(Box::new(OllamaProvider::new()))
}
06 · LoggingProvider

LoggingProvider — Decorator Pattern

Wraps any Arc<dyn LlmProvider>, transparently delegates all calls, and after each successful complete() inserts a row into the token_usage_log PostgreSQL table.

pub struct LoggingProvider {
    inner:     Arc<dyn LlmProvider>,
    pool:      Option<PgPool>,   // None → log silently skipped (for tests)
    operation: Option<String>,   // optional label for the log row
}

// Decorator implementation — fully transparent
#[async_trait]
impl LlmProvider for LoggingProvider {
    fn provider_name(&self) -> &str { self.inner.provider_name() }
    fn model_name(&self) -> &str    { self.inner.model_name() }

    async fn complete(&self, messages, opts) -> Result<LlmResponse> {
        let response = self.inner.complete(messages, opts).await?;
        self.log_usage(&response).await;  // fire-and-forget
        Ok(response)
    }
}

DB Insert (runtime sqlx, no DATABASE_URL at build time)

sqlx::query("INSERT INTO token_usage_log
    (id, provider, model, input_tokens, output_tokens, cost_microdollars, operation)
    VALUES ($1, $2, $3, $4, $5, $6, $7)")
    .bind(uuid::Uuid::new_v4())
    .bind(self.inner.provider_name())
    .bind(&response.model)
    .bind(response.usage.prompt_tokens as i32)
    .bind(response.usage.completion_tokens as i32)
    .bind(estimate_cost(&response.model, response.usage.total_tokens) as i64)
    .bind(self.operation.as_deref())
    .execute(pool).await;
Construction
// Production (with DB pool):
LoggingProvider::new(provider, pool)
  .with_operation("resume-tailor")

// Unit tests (no pool):
LoggingProvider::without_pool(provider)
Design Decisions

Fire-and-forget: let _ = query.execute().await — logging failure is silently swallowed so it never fails the LLM call.

Option<PgPool>: Enables unit testing without a real DB.
