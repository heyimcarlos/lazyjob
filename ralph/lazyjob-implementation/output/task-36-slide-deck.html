Architecture

End-to-End Data Flow

User presses 'r' in JobDetailView | v Action::TailorResume(job_id) | v App::handle_action() --sends--> RalphCommand::Spawn | via mpsc channel v event_loop select! --receives--> RalphCommand | v RalphProcessManager::spawn("resume-tailor", params) | v lazyjob worker subprocess (reads stdin, runs pipeline) | v (stdout NDJSON) WorkerEvent::Status/Results/Done | v pm_rx.recv() in event_loop --maps--> RalphUpdate | v ralph_tx.send(update) | v app.ralph_rx.recv() --> ralph_panel.on_update()

Resume Tailor Handler

  • Connects to DB, loads Job by ID
  • Loads LifeSheet from DB
  • Creates LlmProviderCompleter via Config + CredentialManager
  • Runs ResumeTailor::tailor() with progress channel
  • Maps ProgressEvent → WorkerEvent::Status at each stage
  • Saves ResumeVersion to DB, emits Results + Done

Cover Letter Handler

  • Same DB/profile loading pattern
  • Runs CoverLetterService::generate()
  • Maps CL ProgressEvent → WorkerEvent::Status
  • Emits Results with version_id, template, word count

Event Loop Bridge

crates/lazyjob-tui/src/event_loop.rs

New select! Branches

tokio::select! {
    // ... tick, keyboard events, ralph_rx (existing) ...

    // NEW: Process ralph commands from App
    Some(cmd) = ralph_cmd_rx.recv() => {
        match cmd {
            RalphCommand::Spawn { loop_type, params } => {
                let run_id = process_manager.spawn(&loop_type, params).await?;
                ralph_tx.send(RalphUpdate::Started { id, loop_type });
            }
            RalphCommand::Cancel { run_id } => {
                process_manager.cancel(&run_id).await;
            }
        }
    }

    // NEW: Bridge process manager events to TUI
    result = pm_rx.recv() => {
        if let Ok((run_id, event)) = result {
            let update = map_worker_event_to_ralph_update(run_id, event);
            ralph_tx.send(update);
        }
    }
}

        run_id: id,
        loop_type,           // Was always "unknown" before!

        progress: 0.0,

}

Key Design Decisions

  • mpsc::unbounded channel decouples sync handle_action from async spawn
  • RalphProcessManager lives in event_loop, not App (avoids &mut self async issues)
  • Worker subprocess is same binary (lazyjob worker) for zero-config deployment
  • Each worker loads its own Config + credentials (separate process)
