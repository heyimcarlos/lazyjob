Three Template Strategies

StandardProfessional

Hook opening → Company-specific paragraph → 1-2 achievement paragraphs with metrics → Call to action

ProblemSolution

Open with company challenge → How you solved a similar problem → Bridge to their needs → Call to action

CareerChanger

Acknowledge pivot with confidence → 2-3 transferable skills with evidence → Enthusiasm for new direction → Call to action

CoverLetterGenerator

Prompt Architecture

Each generation produces two prompts:

  • System prompt: Tone description + template-specific structure + anti-cliche rules + anti-fabrication instructions
  • User prompt: Job details (company, title, JD truncated to 1500 chars) + candidate background (top 3 relevant experiences ranked by keyword overlap) + skills + word target

Key Methods

// Static utilities (no LLM needed)

Relevance Ranking

format_relevant_experience() scores each work experience by keyword overlap with the job description, returning the top 3 most relevant entries.

Service Pipeline

CoverLetterService::generate() Flow

10%  [Generating]          LLM call via Completer trait
60%  [CheckingFabrication]  Prohibited phrase detection (20 cliches)
80%  [Persisting]           Version numbering + diff computation + DB save
100% [Done]                 Return CoverLetterVersion

Version Management

  • Monotonic version number per job (MAX(version)+1 via latest_for_job query)
  • Unified diff against previous version using similar::TextDiff
  • First generation: version=1, diff=None
  • Subsequent: version=N+1, diff=unified diff string

Progress Events

Optional mpsc::Sender<ProgressEvent> — if None, all sends are no-ops. Send errors are discarded (closed receiver doesn't halt pipeline).

Layout

Architecture Note

The deeper lazyjob-llm check_grounding() and prohibited_phrase_detector() (35 phrases + LifeSheet grounding) cannot be called from lazyjob-core due to circular dependency. The CLI/TUI layer can run these checks by importing lazyjob-llm directly.

Key Decisions

Why Not Import lazyjob-llm?

lazyjob-llm already depends on lazyjob-core (for LifeSheet types). Adding the reverse dependency would create a circular dependency. Instead, the 20 most common prohibited phrases are checked directly in lazyjob-core. The CLI/TUI layer can run the full 35-phrase + grounding check from lazyjob-llm.

Why similar Crate?

Version diffs are a first-class feature per the spec. similar::TextDiff::from_lines() produces clean unified diffs with configurable context radius. v2.7 is mature and lightweight.

Why gen Was Renamed

Rust edition 2024 reserves gen as a keyword (for generators). All test variables were renamed from gen to generator.

Template-Specific Prompts

Each template gets its own system prompt structure instruction. This avoids a single monolithic prompt and gives the LLM clear structural guidance per template type.

Monotonic Versioning

Version numbers per job use MAX(version)+1 via latest_for_job() query. This is simpler than a separate sequence table and naturally supports the version history UI in the TUI.
