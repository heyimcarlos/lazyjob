FabricationLevel Enum

pub enum FabricationLevel {
    Grounded,     // >= 2 evidence matches from LifeSheet
    Embellished,  // exactly 1 evidence match (partial truth)
    Fabricated,   // 0 evidence matches (hallucinated)
}

Examples

// Grounded: matches company (Acme Corp) + skill (Rust) = 2 evidence
"At Acme Corp, I led the backend engineering team using Rust"

// Embellished: matches skill (Rust) = 1 evidence, but adds unverifiable claims
"Expert Rust developer who architected a revolutionary platform"

// Fabricated: no matches at all
"Founded a blockchain startup that raised $50M"

}

Achievement Matching Detail

Achievement matching uses a word-overlap heuristic: it filters words longer than 3 characters from both the claim and the achievement description, then checks if at least 3 significant words overlap. This catches paraphrased achievements without requiring exact string matches.

// Achievement: "Led migration from Python to Rust reducing latency by 40%"
// Claim:       "Led migration from Python to Rust, cutting latency by 40%"
// Overlap words: "migration", "python", "rust", "reducing/cutting" (3+) -> MATCH

Base64 Detection

Also checks for base64-encoded versions of "system" (c3lzdGVt) and "ignore" (aWdub3Jl) to catch encoding-based evasion attempts. These are matched case-sensitively since base64 is case-sensitive.

Key Design Decisions

DecisionRationale
Evidence count >= 2 for GroundedSingle-match too lenient (a skill name could appear by coincidence). Two corroborating signals = strong evidence.
prompt_injection_guard separate from sanitizerSanitizer is preventive (replaces in template vars). Guard is detective (returns bool on arbitrary input). Different use cases.
validate_grounding returns tuple, not ResultGrounding issues are warnings, not errors. Callers decide whether to reject (Fabricated) or warn (Embellished + prohibited phrases).
No new dependenciesPure string processing. No regex, no NLP crate. Fast and deterministic.
