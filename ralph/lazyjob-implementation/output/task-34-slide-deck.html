Architecture

Where It Fits

The DOCX generator is a pure sync function — no async, no I/O beyond the final file write. It takes the structured ResumeContent (output of the tailoring pipeline) and Basics (from the life sheet) and produces DOCX bytes.

Public API

DocxGenerator

pub struct DocxGenerator;

impl DocxGenerator {
    /// Generate DOCX bytes from resume content and personal info.
    pub fn generate(
        resume: &ResumeContent,
        basics: &Basics,
    ) -> Result<Vec<u8>>;

    /// Generate and save DOCX to a file path.
    pub fn save_to_file(
        resume: &ResumeContent,
        basics: &Basics,
        path: &Path,
    ) -> Result<()>;
}

Usage Example

use lazyjob_core::resume::docx::DocxGenerator;

let (content, _, _) = tailor.tailor(&job, &sheet, opts, None).await?;
let bytes = DocxGenerator::generate(&content, &sheet.basics)?;

// Or save directly:
DocxGenerator::save_to_file(&content, &sheet.basics, Path::new("resume.docx"))?;

Helper Functions

build_contact_line(basics)

Builds pipe-separated contact string from non-None fields.

// All fields: "jane@example.com | +1-555 | https://dev | SF, CA, US"
// Email only: "jane@example.com"
// No fields:  "" (empty string, paragraph skipped)

add_section_heading(doc, title)

Adds bold 12pt section title paragraph.

add_experience_entry(doc, exp)

Renders title/company (bold) + date range (italic) on one line, then bullet points with Unicode • character.

add_education_entry(doc, edu)

Renders "Degree in Field, Year Institution" format.

add_project_entry(doc, proj)

Renders project name (bold, with URL if present), description, and italic technologies line.

format_skills(skills)

Comma-separated primary skills, then "Also: secondary" after pipe separator.

Key Decisions

Use Basics, Not PersonalInfo

The spec mentions a PersonalInfo type but it was never implemented. The existing Basics struct from life_sheet/types.rs already has name, email, phone, url, and location — everything needed for the contact header. No new type necessary.

Cursor for pack()

docx-rs's pack() method requires Write + Seek, not just Write. A raw Vec<u8> doesn't implement Seek, so we wrap it in std::io::Cursor<&mut Vec<u8>>.

Zero-Sized Struct

DocxGenerator is stateless — it's a namespace for pure functions. No configuration or state needed. All formatting constants are module-level const values.

Graceful Empty Handling

Every section is guarded by a non-empty check. An empty ResumeContent produces a valid DOCX containing only the name header. No panics, no errors.
