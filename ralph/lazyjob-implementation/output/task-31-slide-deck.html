Template Engine

SimpleTemplateEngine performs single-pass {variable} interpolation.

Interpolation Rules

  • {varname} is replaced with the value from TemplateVars
  • Missing variable returns TemplateError::MissingVariable
  • Unmatched { without closing } passes through verbatim (safe for JSON in prompts)
  • Empty values are valid (empty string replacement)
let engine = SimpleTemplateEngine;
let vars = template_vars! {
    "companies" => "Stripe, Anthropic",
    "skills" => "Rust, Python",
};
let rendered = engine.render(
    registry.get(LoopType::JobDiscovery)?,
    &vars
)?;
// rendered.system → fully interpolated system prompt
// rendered.user → fully interpolated user prompt
let messages = rendered.into_chat_messages();
let response = provider.complete(messages, opts).await?;

DefaultPromptRegistry

Loads all 9 embedded TOML templates at initialization. Supports user overrides from disk.

API

// Load all 9 embedded templates
let mut registry = DefaultPromptRegistry::new()?;

// Get a specific template
let tmpl = registry.get(LoopType::JobDiscovery)?;

// List all templates
let all = registry.all(); // Vec<&PromptTemplate>, sorted by LoopType

// Override a template (testing)
registry.override_template(LoopType::JobDiscovery, custom_template)?;

// Load user overrides from ~/.config/lazyjob/prompts/*.toml
let count = registry.load_user_overrides(config_dir)?;

User Overrides

Users can place custom .toml files in ~/.config/lazyjob/prompts/ to override any embedded template. Each override file must be a valid PromptTemplate TOML. The loop_type field determines which embedded template gets replaced.

Anthropic Prompt Caching

build_anthropic_system_field() injects cache_control for ~90% cost reduction on repeated system prompts.

With Caching (default)

// When cache_system_prompt = true (default for all templates except ErrorResponse):

  {
    "type": "text",
    "text": "You are a job search assistant...",
    "cache_control": { "type": "ephemeral" }
  }
]

Without Caching

// When cache_system_prompt = false:
"You are an error recovery assistant..."  // plain string

Key Design Decisions

  • TOML + include_str! (no runtime file loading, no Tera/Handlebars dependency)
  • Single-pass {variable} interpolation (simple, fast, handles JSON safely)
  • template_vars! macro enforces sanitization at the call site
  • RenderedPrompt::into_chat_messages() bridges templates to LlmProvider
  • Typed output structs with serde validation + custom business rules
