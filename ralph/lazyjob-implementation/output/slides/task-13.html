Reuse of centered_rect: Rather than duplicating the centering logic, ConfirmDialog imports centered_rect from modal_dialog (it's a pub fn). This keeps the centering behavior consistent across all modal-style widgets.

Bug Fix: centered_rect Height Overflow

Root cause: When popup height > terminal height (e.g., a 9-row dialog on a 5-row terminal), centered_rect returned a Rect with height = 9 but y-position = 0, extending to row 8 — outside the 5-row buffer. ratatui panicked with "index outside of buffer".
crates/lazyjob-tui/src/widgets/modal_dialog.rs
pub fn centered_rect(percent_x: u16, height: u16, r: Rect) -> Rect {
+ let clamped_height = height.min(r.height);
let vertical = Layout::default()
.direction(Direction::Vertical)
.constraints([
Constraint::Fill(1),
- Constraint::Length(height),
+ Constraint::Length(clamped_height),
Constraint::Fill(1),
])
.split(r);
let w = (r.width * percent_x / 100).max(1).min(r.width);
let x = r.x + (r.width.saturating_sub(w)) / 2;
- Rect::new(x, vertical[1].y, w, height)
+ Rect::new(x, vertical[1].y, w, clamped_height)
}
Fix: Clamp height to r.height before both the Layout constraint and the final Rect construction. The Layout solver then produces valid y-coordinates within bounds, and the output Rect height equals the clamped value. This makes all dialog widgets (ModalDialog, ConfirmDialog) safe on tiny terminals.
Note: This bug existed in modal_dialog.rs before this task but its test (modal_dialog_small_area_does_not_panic) was never running because pub mod widgets was absent from lib.rs. Adding the module declaration exposed the latent failure.
