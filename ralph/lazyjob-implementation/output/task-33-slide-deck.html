Architecture

Circular Dependency Solution

lazyjob-llm depends on lazyjob-core (for LifeSheet types). So lazyjob-core cannot depend on lazyjob-llm.

Solution: Reuse the existing Completer trait from lazyjob_core::discovery::company:

#[async_trait]
pub trait Completer: Send + Sync {
    async fn complete(&self, system: &str, user: &str) -> Result<String>;
}

All LLM calls in the resume pipeline go through this trait. The CLI/Ralph layer bridges LlmProvider to Completer.

JD Parser

LLM-Backed (Primary)

Sends JD to LLM with structured JSON system prompt. Extracts:

  • Required skills with canonical (lowercase) names
  • Nice-to-have skills
  • Keywords for bullet rewriting
  • Responsibilities for summary generation

Handles preamble text and markdown fences via extract_json().

Regex Fallback

Activated automatically when LLM fails. Uses section header detection (Requirements/Nice to Have/Responsibilities) and 26 tech keyword patterns.

let jd = match self.jd_parser.parse(raw_jd).await {
    Ok(jd) => jd,
    Err(_) => self.fallback_parser.parse_sync(raw_jd)?,
};

Content Drafter

Summary Generation (1 LLM call)

Generates 3-sentence summary using matched skills and JD responsibilities. Anti-cliche system prompt.

Bullet Rewriting (1 LLM call per experience)

Sends original bullets + target keywords. Returns JSON array of rewritten bullets. Falls back to originals on parse failure.

Skills Ordering (No LLM)

Sorts skills into primary (JD-matched) and secondary. No LLM call needed.

Fabrication Auditor

Skill Risk Assessment

  • Forbidden: Claims certifications not in LifeSheet (PMP, CISSP, CPA, etc.)
  • High: No evidence in skills, tech_stack, or projects
  • Low: Fuzzy match (Jaro-Winkler ≥ 0.88)
  • None: Exact match in LifeSheet

Unsupported Claim Detection

For rewritten bullets, extracts numeric claims (N%, Nx, $N, NK/NM) and checks against original achievement metrics. Custom char-by-char parser, no regex dependency needed.

"Improved throughput by 500%" -- 500% not in life sheet -> WARNING
"Reduced latency by 40%"     -- 40% matches life sheet -> OK

Key Decisions

  • Completer trait reuse: Used existing discovery::Completer to avoid circular dependency. Same pattern as CompanyResearcher.
  • No regex crate: Custom char-by-char numeric parser for claim detection. Avoids adding dependency for simple pattern matching.
  • JSONB storage: All report data stored as JSONB in PostgreSQL for flexible querying and schema evolution.
  • Automatic fallback: LLM JD parser falls back to regex parser transparently. Bullet rewriting falls back to original bullets on LLM failure.
  • Progress events optional: Option<mpsc::Sender> pattern - callers not using TUI can pass None.
  • Pre-check + post-check: Fabrication auditor runs twice - once on raw LifeSheet (pre-check) and once on generated content (post-check).
